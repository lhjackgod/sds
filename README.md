# Prompt-aware SMPL-X Garment UV Masks

Phase 1 generates body-aligned garment UV masks from a prompt and an SMPL-X
template mesh. It intentionally does not implement SDS, CasTex optimization, or
garment geometry for skirts, dresses, open jackets, capes, and scarves.

## Inputs

Place licensed SMPL-X source assets and generated files under `data/smplx/`:

```text
data/smplx/
  source/                             # manually downloaded licensed inputs
    models/
      smplx/
        SMPLX_NEUTRAL.npz
    smplx_uv_template.obj             # official UV template OBJ
  generated/                          # generated by the script below
    smplx_template.obj
    uv_data.npz
    part_labels.json
```

The SMPL-X model parameter file and official UV template are not redistributed
by this repository. The OBJ must contain vertices and triangular faces. UV
coordinates can come from OBJ `vt` entries or from an NPZ with:

```text
uv_coords:       [N_uv, 2]
face_uv_indices: [F, 3]
```

`part_labels.json` can be a JSON list or an object containing a
`vertex_labels` or `labels` list. There must be one label per mesh vertex.

## Generate the SMPL-X Template

The generator runs SMPL-X with zero-valued parameters, copies UV coordinates
from the official UV OBJ, and converts PARTE's SMPL-X segmentation labels into
the garment-oriented labels used by this project:

```bash
/root/miniconda3/envs/castex/bin/python src/generate_smplx_template.py \
  --model-path data/smplx/source/models \
  --uv-template data/smplx/source/smplx_uv_template.obj \
  --out data/smplx/generated
```

The script first tries an installed `smplx` package. If it is unavailable, it
uses the compatible implementation vendored in `../PARTE_RELEASE`. By default,
the source segmentation is read from:

```text
../PARTE_RELEASE/data/body_data/smplx_vert_segmentation.json
```

PARTE's segmentation leaves six head-area vertices unlabeled. The generator
emits a warning and explicitly assigns those vertices to `head`, which keeps
them in the skin region. You can omit `--uv-template` to generate geometry and
labels only. UV mask generation still requires UV data.

## Generate UV Masks

```bash
/root/miniconda3/envs/castex/bin/python src/main_generate_masks.py \
  --prompt "a person wearing a red short sleeve T-shirt and blue jeans" \
  --mesh data/smplx/generated/smplx_template.obj \
  --part-labels data/smplx/generated/part_labels.json \
  --out outputs/masks/test_001 \
  --resolution 1024
```

The generated OBJ already contains UV entries. Add
`--uv data/smplx/generated/uv_data.npz` if you want to load the generated UV
NPZ explicitly. Outputs
include hard and soft masks, `combined_region_map.png`, `debug_texture.png`,
and `debug_mesh.obj` with its MTL file. Open `debug_mesh.obj` in Blender or
MeshLab to inspect the UV regions on the body.


## Batch Prompt Debug

After `data/smplx/generated/` exists, generate the five default prompt cases with:

```bash
/root/miniconda3/envs/castex/bin/python src/batch_generate_prompt_masks.py \
  --resolution 512 \
  --out-root outputs/masks/prompt_batch_512
```

Each prompt directory contains three visual checks:

```text
debug_texture.png             postprocessed UV region texture with boundary in yellow
raw_vertex_debug_texture.png  UV texture before postprocess, directly from vertex majority vote
vertex_region_mesh.ply        3D vertex-colored mesh for checking body-part segmentation
```

Color legend:

```text
gray   = skin
red    = upper garment
blue   = lower garment
yellow = clothing boundary, only in debug_texture.png
black  = invalid/background UV area
```

Open `debug_mesh.obj` to inspect the postprocessed UV texture on SMPL-X. Open
`vertex_region_mesh.ply` to inspect whether the vertex-level body labels and
prompt rules are correct before UV rasterization.

## Labels

Expected labels:

```text
head neck torso pelvis
left_upper_arm right_upper_arm left_forearm right_forearm
left_hand right_hand left_thigh right_thigh left_calf right_calf
left_foot right_foot
```

Shorts use the upper half of thigh vertices. Configure the body vertical axis
in `configs/garment_rules.yaml` when the template does not use Y-up.


## Phase 2: SO-SMPL-inspired Bounded Garment Offset

Phase 2 borrows the SO-SMPL idea of freezing the human body and representing
body-aligned clothing as a bounded shell. In this project the shell is a scalar
normal offset sampled from a UV offset map:

```text
v_offset = v_body + s(v) * normal(v)
```

The offset is restricted by Phase 1 garment masks, max-thickness rules, smoothing,
and fixed zero regions. It is a garment structure prior for later CasTex or
text-to-texture, not a true separated garment reconstruction.

Limitations:

```text
- only body-aligned garments are supported
- offset is bounded normal offset, not arbitrary XYZ displacement
- skirt, dress, open jacket, cape, scarf are not supported
- head, hands, feet, and skin remain fixed at zero offset
- SDS refinement is optional and constrained by masks and max offset
- without diffusion dependencies, mode=init and --sds-mode dummy still run
```

Generate only rule-initialized offset:

```bash
/root/miniconda3/envs/castex/bin/python src/main_generate_offset.py \
  --prompt "a person wearing a red short sleeve T-shirt and blue jeans" \
  --mesh data/smplx/generated/smplx_template.obj \
  --part-labels data/smplx/generated/part_labels.json \
  --mask-dir outputs/masks/prompt_batch_512/01_a_person_wearing_a_red_short_sleeve_t_shirt_and_blue_jeans \
  --out outputs/offset/test_tshirt_jeans_init \
  --mode init
```

Run the SDS refinement scaffold with dummy guidance:

```bash
/root/miniconda3/envs/castex/bin/python src/main_generate_offset.py \
  --prompt "a person wearing a loose white hoodie and dark blue jeans" \
  --mesh data/smplx/generated/smplx_template.obj \
  --part-labels data/smplx/generated/part_labels.json \
  --mask-dir outputs/masks/prompt_batch_512/04_a_person_wearing_a_white_hoodie_and_dark_jeans \
  --out outputs/offset/test_hoodie_jeans_dummy_sds \
  --mode sds \
  --optimize-mode part_scale \
  --steps 300 \
  --sds-mode dummy
```

Each offset run writes:

```text
offset_mesh.obj
offset_scale.npy
offset_scale_uv.png
offset_debug_texture.png
offset_vertex_colors.ply
render_body_front.png
render_offset_front.png
render_normal_front.png
render_silhouette_front.png
optimization_log.json
```

`offset_scale_uv.png` is the scalar offset map visualized in UV space.
`offset_vertex_colors.ply` shows upper/lower/skin vertices with brightness
modulated by offset strength. `optimization_log.json` records the prompt,
GarmentSpec, mode, loss curves, final offset statistics, and part scale metadata.
